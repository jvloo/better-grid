# Getting Support

OSS-first project with source-available pro plugins, maintained by a small team. No SLAs, but every channel is read.

## Where to ask

| You have…                                  | Go to                                                                                       |
| ------------------------------------------ | ------------------------------------------------------------------------------------------- |
| A bug                                      | [Open a bug report](https://github.com/jvloo/better-grid/issues/new?template=bug_report.yml) |
| A feature idea                             | [Open a feature request](https://github.com/jvloo/better-grid/issues/new?template=feature_request.yml) |
| A "how do I…?" question                    | [Start a Discussion](https://github.com/jvloo/better-grid/discussions)                       |
| A migration question (AG Grid, MUI X, …)   | [`docs/migrations/`](docs/migrations/)                                                       |
| A Pro license / commercial-use question    | Email **ping@xavierloo.com**                                                                 |
| A security finding                         | [`SECURITY.md`](SECURITY.md). **Do not open a public issue.**                                |
| Something interesting you built            | Share it in the "Show & tell" Discussions category.                                          |

## Before you open an issue

- Reproduce on the latest 1.x. Older minors don't get patches.
- Search closed issues — many oddities have been hit before.
- Trim your repro. The Vite playground at `apps/playground` is a good template — fork it, reproduce, link the fork.

## Response expectations

- **Bugs with a clear repro:** triaged within a week, often within 48 hours.
- **Feature requests:** read promptly, queued against [`ROADMAP.md`](ROADMAP.md).
- **Discussions questions:** answered when a maintainer is around; community help welcome.

## Commercial use and support

`@better-grid/pro` source-available; commercial production use requires a Pro license. For a license, faster-than-OSS feedback, guaranteed response times, or a custom-built plugin: **ping@xavierloo.com**.

## Sponsorship

Independent project. Support continued development via **sponsors@xavierloo.com** or the GitHub Sponsors button (when enabled). Sponsorship is appreciation; it does not grant a Pro license.
